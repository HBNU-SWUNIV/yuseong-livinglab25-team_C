## 💡 Project Background

🏷️ **서비스명**: **"유성안심문자(Yuseong Care SMS)"**

디지털 취약계층, 특히 고령층에게 문자(SMS)를 통해 생활 필수 정보를 자동으로 제공하는 서비스.

스마트폰이나 별도의 앱 설치 없이도 피처폰으로 날씨, 미세먼지, 폭염·한파 경보, 보건소 공지 등 필수 정보를 받을 수 있도록 합니다.

> 🌟 **핵심 가치: "앱이 없어도, 스마트폰이 없어도, 문자 한 통으로 안심하는 하루"**

## 🤔 필요성

* 디지털 취약계층(고령층)은 정보를 전달받을 수 있는 수단이 매우 제한적.
* 현장 방문(경로당) 및 면담 결과, 이미 보급된 스마트 키오스크 등 디지털 기기들은 사용법이 익숙지 않아 대부분 방치.
* 어르신들은 새로운 기술보다 **기존에 익숙한 방식(전화, 문자)을 압도적으로 선호**. 📞
* 실제로 피처폰 사용률이 여전히 높으며, 스마트폰 소지자도 전화/문자 등 기본 기능만 사용하는 경우가 대부분.
* 따라서 고령층에게 가장 익숙한 **문자(SMS)**를 기반으로 필수 정보를 전달하는 서비스가 필요. 📬

## 🚫 기존 해결책의 문제점

현재 유성구에서 운영 중인 '유성별별 문자 알림 서비스'가 있으나, 몇 가지 한계점이 존재.

* **타겟 불일치**: 모든 구민을 대상으로 하기에 고령층에게 특화된 필수 정보(안전, 건강, 복지)가 아닌 구정 소식, 문화 행사 등 일반 정보 위주로 제공됨.
* **접근성 문제**: 서비스 신청을 홈페이지에서 개인이 직접 등록해야 하므로, 디지털 기기 사용이 어려운 고령층이 스스로 가입하기에 어려움 존재.

본 '유성안심문자' 서비스는 이러한 문제를 해결하기 위해 **고령층을 주 대상**으로 하며, **날씨, 안전, 복지 등 생활 밀착형 정보**를 중심으로 제공합니다.

## 🏗️ System Design



### 시스템 레이어 구조 📊

1.  **프레젠테이션 레이어**: 관리자 웹 패널 (React/Vue.js)
2.  **비즈니스 로직 레이어**: 백엔드 API 서버 (Node.js/Spring Boot)
3.  **데이터 레이어**: 데이터베이스 (MySQL/Firebase)
4.  **외부 통합 레이어**: 공공 API 및 SMS 게이트웨이 연동
5.  **스케줄링 레이어**: 정기 작업 및 모니터링

## 📋 System Requirements

주요 기능 구성은 다음과 같습니다.

* ☀️ **자동 문자 발송 (기본형)**
    * 매일 오전 7시, 오늘의 날씨/기온/미세먼지/특보를 자동으로 발송.
* 🚨 **긴급 재난 알림 (즉시형)**
    * 한파, 폭염, 지진 등 재난 상황 발생 시 기상청 API와 연동하여 즉시 대상자에게 긴급 알림을 전송.
* 🏥 **보건·복지 알림 (확장형)**
    * 유성구 보건소, 복지과에서 관리자 패널을 통해 등록한 일정(예: 독감 예방접종, 경로당 행사)을 예약 발송.
* 💊 **개인 맞춤형 알림 (선택형)**
    * 본인 혹은 가족이 관리자 패널을 통해 개인별 알림(예: "혈압 측정일", "의약품 복용 시간")을 설정.

## 📱 Case Study (발송 예시)

### 1. 🌤️ 평상시 (일상 발송)

> "안녕하세요! 오늘 유성구의 날씨는 맑고 10~18도입니다. 미세먼지 '보통' 외출시 마스크를 챙기세요. 유성구청 드림 -"

### 2. 🔥 폭염특보 시 (긴급)

> “[폭염주의보] 오늘 낮 최고 34도 예상! 야외활동 자제, 물 자주 마시세요. 유성구 안전재난과 -“

### 3. 💉 복지 알림

> "11월 10일 유성구 보건소에서 독감 무료접종이 있습니다. 65세 이상 어르신은 신분증 지참해주세요."

## 🛠️ Description (기술 구성)

| 구분 | 기술 | 비고 |
| :--- | :--- | :--- |
| **백엔드** | Node.js / Spring Boot | 스케줄러 + API 통합 |
